# Spec: Generate Invoice

## Overview

This feature adds a printable invoice/quotation page for an enquiry at
`GET /enquiries/<id>/invoice`. It renders a standalone HTML page (no base.html
navbar or sidebar) styled for print, showing the company's profile as the
seller, the enquiry's customer as the buyer, and the full particulars table
as line items with subtotals and a grand total. A "Print / Save PDF" button
triggers the browser's native print dialog. The invoice document is a formal
quotation derived from the enquiry's particulars ŌĆö it is separate from the
shipment-level `bill_print` feature. A "Generate Invoice" button is added to
the action bar in `enquiry_detail.html` so users can reach this page in one
click.

## Page layout ŌĆö enquiry_invoice.html

The page is a standalone HTML document (no `{% extends %}`). It follows the
same structural pattern as `bill_print.html`.

### Top bar (`.bill-actions.no-print`)
- "Print / Save PDF" button ŌĆö `onclick="window.print()"`
- "ŌåÉ Back to Enquiry" link ŌĆö `url_for('enquiry_detail', enquiry_id=enq.id)`

### Invoice page (`.bill-page`)

**1. Header (`.bill-header`)**

Left column ŌĆö seller (company profile):
- Company name, legal name, address lines, city/state/country/pincode
- GST, PAN, IEC if set
- Email and phone if set
- Falls back to `user.name` / "LogiTrack Account" if no company profile

Right column ŌĆö invoice metadata (`.bill-title-block`):
- Title: `QUOTATION` (large, prominent)
- Enquiry No. (`.bill-meta-row`)
- Print Date (today's date)
- Enquiry Date
- Status badge

**2. Customer / Bill-To section (`.bill-section`)**
- Section title: "Bill To"
- Customer Name, Email, Phone ŌĆö sourced from `enq.customer_name`, `enq.customer_email`, `enq.customer_phone`
- If all three are empty, show "Customer details not provided"

**3. Cargo Details section (`.bill-section`)**
- Section title: "Cargo Details"
- Two-column keyŌĆōvalue grid (`.bill-kv-grid`) showing:
  Commodity, Consignment Type, Shipment Terms, Weight (with unit), Packages,
  MAWB, HAWB, Route (origin ŌåÆ destination), Incoterms, Currency, Ex.rate
- Skip any field that is `None` / empty

**4. Particulars / Line Items section (`.bill-section`)**
- Section title: "Particulars"
- If particulars exist: a table (`.bill-items-table`) with columns:
  `#`, Particular, SAC/HSN, QTY, Weight, Offered Rate, Expense, Tax %, CGST, SGST, IGST, **Total**
- `<tfoot>` grand-total row spanning all non-total columns, bold total value
- Currency label above or beside the total (e.g. `INR 12,345.00`)
- If no particulars: plain text "No particulars on this enquiry."

**5. Notes section (`.bill-section`)** ŌĆö only if `enq.notes` is non-empty
- Section title: "Notes"
- Plain `<p>` block

**6. Footer (`.bill-footer`)**
- "Generated by LogiTrack" and today's date ŌĆö centred, small, muted

## enquiry_invoice.css

Reuse the same CSS class names as `bill_print.css` where possible
(`.bill-page`, `.bill-header`, `.bill-from`, `.bill-title-block`,
`.bill-section`, `.bill-section-title`, `.bill-actions`, `.bill-footer`,
`.no-print`). Add new classes only for parts that differ (e.g.
`.bill-kv-grid`, `.bill-items-table`).

`@media print` block must:
- Hide `.no-print`
- Remove margins/shadows from `.bill-page`
- Ensure the table does not break across pages if possible (`page-break-inside: avoid`)

Use CSS variables (`--ink`, `--ink-muted`, `--accent`, `--surface`, `--border`,
`--radius`, etc.) ŌĆö never hardcode hex values.

## "Generate Invoice" button in enquiry_detail.html

Add the button in the `.enq-detail-action-bar` div, after the existing
"Manage Particulars" button. It should always be visible (not gated on status):

```html
<a href="{{ url_for('enquiry_invoice', enquiry_id=enq.id) }}"
   class="enq-action-btn" target="_blank">Generate Invoice</a>
```

`target="_blank"` opens the invoice in a new tab, keeping the detail page open.

## app.py route outline

```
GET /enquiries/<int:enquiry_id>/invoice
  ŌåÆ login check (redirect to login if not logged in)
  ŌåÆ get_enquiry_by_id(enquiry_id) ŌĆö abort(404) if not found
  ŌåÆ ownership check (enq["user_id"] != session["user_id"]) ŌĆö abort(403)
  ŌåÆ get_particulars_by_enquiry(enquiry_id)
  ŌåÆ grand_total = sum(p["total"] for p in particulars)
  ŌåÆ company = get_company_profile(user_id)
  ŌåÆ today = date.today().isoformat()
  ŌåÆ render_template("enquiry_invoice.html",
        user=user, enq=enq,
        particulars=particulars, grand_total=grand_total,
        company=company, today=today)
  (no active_section ŌĆö standalone page, no sidebar)
```

## Rules for implementation

- No SQLAlchemy or ORMs ŌĆö use `get_enquiry_by_id`, `get_particulars_by_enquiry`, `get_company_profile` from `database/db.py`
- Parameterised queries only ŌĆö never f-strings in SQL
- Passwords hashed with werkzeug (not applicable here)
- Use CSS variables ŌĆö never hardcode hex values
- `enquiry_invoice.html` must **not** extend `base.html` ŌĆö it is a standalone print document, same pattern as `bill_print.html`
- Ownership check: verify `enq["user_id"] == session["user_id"]` and `abort(403)` if not
- Do not add new pip packages
- `target="_blank"` on the "Generate Invoice" button in `enquiry_detail.html`
